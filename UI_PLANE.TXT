Phase 2 - Shared / Reusable Component Architecture

IMPORTANT: ~929 pages exist across 11 dashboards but many modules repeat (Attendance, Reports, Payroll, Settings appear in 4-5 dashboards).
To avoid building 929 separate pages, use a shared component architecture.

Reusable Page Templates (15-20 Templates)

1. List Page Template
   - Used For: All Students, All Employees, All Books, All Vehicles, All Routes etc.
   - Components: Search Bar, Filters, Data Table, Pagination, Bulk Actions, Export Button
   - Reused in: ~80+ pages across all dashboards

2. Add/Edit Form Template
   - Used For: Add Student, Add Employee, Create Fee, Add Vehicle etc.
   - Components: Form Fields, Validation, File Upload, Step Wizard (for large forms), Submit/Cancel
   - Reused in: ~60+ pages

3. Detail/Profile Page Template
   - Used For: Student Profile, Employee Profile, Vehicle Detail, Book Detail etc.
   - Components: Header Card, Tab Navigation, Info Sections, Activity Timeline, Related Items
   - Reused in: ~30+ pages

4. Dashboard Overview Template
   - Used For: All 11 Dashboard home pages
   - Components: KPI Cards Row, Charts Grid (2-3 columns), Recent Activity List, Quick Actions Panel
   - Reused in: 11 pages

5. Report Page Template
   - Used For: All report pages (Student Reports, Fee Reports, Attendance Reports etc.)
   - Components: Date Range Picker, Filters, Report Table, Charts, PDF Export, Excel Export, Print
   - Reused in: ~100+ pages

6. Analytics Page Template
   - Used For: Fee Analytics, Attendance Analytics, Performance Analytics etc.
   - Components: KPI Cards, Multiple Charts, Comparison Tables, Trend Lines, Forecasting Widget
   - Reused in: ~20+ pages

7. Settings Page Template
   - Used For: All Settings pages across every dashboard
   - Components: Settings Groups, Toggle Switches, Input Fields, Save/Reset Buttons
   - Reused in: ~40+ pages

8. Approval/Request Page Template
   - Used For: Leave Requests, Fee Discounts, Scholarships, Refunds, Expense Approval etc.
   - Components: Request List, Status Badges, Approve/Reject Buttons, Comments, History Timeline
   - Reused in: ~25+ pages

Phase 3 - Responsive / Mobile Design

All pages must be responsive. These breakpoints will be used:

Breakpoints
- Mobile: 320px - 640px
- Tablet: 641px - 1024px
- Desktop: 1025px - 1440px
- Large Desktop: 1441px+

Mobile Behavior
- Sidebar: Hidden by default, opens as overlay drawer on hamburger menu tap
- Header: Sticky top, compact mode (hide branch selector text, show icons only)
- KPI Cards: Stack vertically (1 per row on mobile, 2 per row on tablet)
- Data Tables: Horizontal scroll with sticky first column, or switch to card view
- Charts: Full width, stack vertically
- Forms: Single column layout
- Quick Actions: Horizontal scrollable row or grid (2 columns)
- Modals: Full screen on mobile

Tablet Behavior
- Sidebar: Collapsed to icon-only mode, expands on hover
- KPI Cards: 2-3 per row
- Data Tables: Horizontal scroll if needed
- Charts: 2 per row
- Forms: 2 column layout

Desktop Behavior
- Sidebar: Expanded by default, collapsible
- KPI Cards: 4-5 per row
- Data Tables: Full width
- Charts: 2-3 per row as shown in mockups
- Forms: 2-3 column layout

Touch Friendly
- Minimum touch target: 44px x 44px
- Swipe gestures for sidebar open/close
- Pull to refresh on list pages
- Bottom navigation bar for most used actions on mobile

Phase 10 - Accessibility (a11y) Standards

Accessibility is important for all users including differently-abled staff and students.

Keyboard Navigation
- All interactive elements focusable via Tab key
- Skip to main content link
- Keyboard shortcuts for common actions
- Focus visible outline on all interactive elements
- Escape key to close modals and dropdowns

Screen Reader Support
- Proper ARIA labels on all interactive elements
- ARIA roles for custom components (sidebar, tabs, modals)
- Alt text for all images
- Descriptive link text (not "click here")
- Form field labels properly associated
- Error messages announced to screen readers
- Live regions for dynamic content updates (ARIA-live)

Color and Contrast
- WCAG 2.1 AA minimum contrast ratio (4.5:1 for text, 3:1 for large text)
- Do not rely on color alone to convey information (use icons + text + color)
- Status badges use icons along with colors (checkmark for success, X for error)
- Chart legends include patterns/shapes in addition to colors

Text and Typography
- Base font size: 16px minimum
- Line height: 1.5 minimum
- Allow browser zoom up to 200% without layout breaking
- No text in images (except logos)

Motion and Animation
- Respect prefers-reduced-motion OS setting
- Provide option to disable animations in Settings
- No flashing content (seizure risk)
- Loading states use skeleton screens instead of spinners where possible
